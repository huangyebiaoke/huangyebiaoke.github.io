我的个人博客搭建历程:

我在2017/06/20开始接触到独立博客,当时觉得很新鲜,独立博客上的大牛写的文章又好看又有技术含量,觉得博主们就像世外的高人一般;而且一个一个的友链就好像一张网,连接着每个个人博客,形成了一个博客圈子,我呆在独立博客的地方出不去了.

  1. 当时便下定决心自己也要搞一个独立博客,开始用的是LAMP(Linux+Apache+MySQL+PHP)结合Wordpress在我的Windows10笔记本的WSL(Windows Linux Subsystem)搭建的博客系统,然后通过ngrok.cc内网穿透实现公网访问[1].


  2. 用过一段时间后觉得WordPress方便是方便,就是插件太多,太臃肿了,而且服务开在本地我的笔记本不能时时刻刻开着机,要不然就是神船也得变沉船;遂在aliyun租了一台ECS,转向另一个博客框架Typecho,我还记得Typecho官网那句念念不忘,必有回响.
  3. aliyun的ECS虽然学生价低是低(100多RMB),但考虑到我迟早成为社会人之后肯定不能每年都续租.于是用Google搜索了一下有没有免费的博客平台[2].
  4. 我看到了hexo+github page的相关文章,遂脱离typecho转向静态博客的怀抱(不用担心被DDOS[3]).
    1. 一开始用的主题是自己魔改的yilia主题.
    2. 用了大概一年左右,产出大概10篇左右的博文[4],那段风雨飘摇的年代Github被DNS污染及GFW墙,我先后辗转于code page,gitee page,qiniu静态资源托管,感受到小人物没钱的无奈与心酸.
    3. 关于评论系统我一开始用的是畅言(搜狐的免费社会化评论系统),大概在18年末的时候挂了;所以切换到gitalk,但由于GFW和DNS污染,通常出现network err的警告;我就切换回国内的评论系统valine(也是Pinghsu主题自带的).
    4. 2020年年初,考完研重新启程,我又回到了Github Page的怀抱,用qiniu CDN动态加速国内访问速度;觉得写作应该focus on文章,所以想找一个素一点的主题,符合极简的设计,于是就找到了Pinghsu,之后自己再改了一些布局和css,让其match自己的审美.
  1. 1.期间看过一个叫N2N项目的论文,它是通过UDP打孔来进行一对一的内网穿透,不需要外网服务器;不过这个作者弃坑了,所以软件很不稳定 ↩
  2. 2.简书,掘金,思否不考虑;因为那时候我水平太低,只想做一个安安静静的美男子 ↩
  3. 3.做JSP作业的时候有过被同学D的经历(用JSP的Application关键字存储聊天信息,通过每5s http轮询刷新页面实现聊天室的功能),不过没换IP所以很容易用Linux找出访问频繁的IP,然后Ban掉 ↩
  4. 4.我那时知道自己的水平,给自己的网站slogan是科普计算机小知识 ↩
